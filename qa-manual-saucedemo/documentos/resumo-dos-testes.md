# Resumo dos Testes Executados

## Projeto

Testes manuais na aplicação SauceDemo.

## Objetivo

Validar os principais fluxos da aplicação, simulando uma rotina inicial de QA Manual com organização das atividades no Jira.

## Funcionalidades testadas

### 1. Login

Foram validados os seguintes cenários:

- login com credenciais válidas
- login com senha incorreta
- login com campos vazios
- login com usuário bloqueado

### 2. Exibição de produtos após o login

Foram verificadas:

- exibição da listagem de produtos
- nome dos produtos
- descrição
- preço
- imagem
- botão de adicionar ao carrinho

### 3. Carrinho de compras

Foram validadas as ações de:

- adicionar produto
- remover produto
- atualização do contador de itens

### 4. Fluxo de checkout

Foram validadas as etapas de:

- iniciar checkout
- preencher dados obrigatórios
- revisar informações da compra
- concluir pedido

## Resultado final

Todos os cenários executados apresentaram o comportamento esperado durante os testes.

Nenhum defeito foi identificado nos fluxos validados nesta execução.

## Evidências coletadas

Foram registradas evidências visuais das principais etapas do projeto, incluindo:

- quadro Kanban no Jira
- tela de produtos após autenticação
- carrinho de compras
- revisão do checkout
- checkout finalizado
